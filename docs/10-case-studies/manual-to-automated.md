# Migrating from Manual to Automated KYC

## Overview

Phased migration from manual KYC (branch-based, paper forms) to automated eKYC — change management, parallel running, and measuring success.

---

## Migration Phases

| Phase | Activities | Duration |
|-------|-----------|----------|
| **1. Digitize** | Scan paper, create digital records | 3-6 months |
| **2. Assist** | Digital tools assist manual reviewers | 3-6 months |
| **3. Automate** | Auto-approve clear cases, manual for borderline | 6-12 months |
| **4. Optimize** | Tune thresholds, reduce manual review rate | Ongoing |

## Critical Success Factors

| Factor | Why It Matters |
|--------|---------------|
| **Regulatory buy-in** | Get regulator approval before deployment |
| **Staff training** | Agents must understand new workflow |
| **Parallel running** | Run manual + automated side-by-side to validate |
| **Gradual rollout** | Start with low-risk products, expand progressively |
| **Metrics dashboard** | Real-time visibility into automated vs manual outcomes |

---

## Key Takeaways

!!! success "Summary"
    - Migration takes **12-24 months** for large institutions — don't rush
    - **Parallel running** builds confidence — compare automated decisions with manual outcomes
    - **Staff retraining** (not just replacement) is critical — manual reviewers become exception handlers
